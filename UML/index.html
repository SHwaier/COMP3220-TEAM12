Phase 3

1. Singleton Pattern (DataManager)

Usage: Ensures a single instance of the DataManager class across the entire application.

Implementation: A private static instance of RecordsCentral and a public static getInstance() method to access the single instance.

Purpose: Streamlined and centralized access to resources across the app, improving data access and reducing redundancy.

2. Factory Pattern (RecordFactory)

Usage: Creates instances of the Record class.

Implementation: RecordFactory.createRecord(String year, String place, int amount) validates inputs and encapsulates the creation logic for Record objects.

Purpose: Centralizes object creation and enforces validation to ensure consistency and reusability.

3. Strategy Pattern (FilterManager and Filters)

Usage: Dynamically applies different filters (e.g., YearFilter, RegionFilter) to the data.

Implementation:

  • Filter interface defines the apply method.
  • Concrete implementations like YearFilter and RegionFilter provide specific filtering logic.
  • FilterManager aggregates and applies multiple filters dynamically.

Purpose: Provides flexibility to add or combine filters without modifying the core logic, adhering to the Open/Closed Principle.

4. Composite Pattern (RecordsList)

Usage: Encapsulates a collection of Record objects and provides operations to manipulate and retrieve subsets of the data.

Implementation: RecordsList acts as a container for multiple Record objects and provides methods like getRecordsForYear and getRecordsForPlace.

Purpose: Treats collections of objects uniformly, simplifying operations on grouped data.

5. Adapter Pattern (UI Filters Integration)

Usage: Integrates MultiSelectComboBox and filter logic with the FilterManager for seamless application of user selections.

Implementation: Adapts the multi-select UI inputs into a form usable by the FilterManager, such as converting selected items into RegionFilter or YearFilter for application on RecordsList.

Purpose: Bridges the gap between the UI layer and the business logic, ensuring compatibility.
